## 1. 제품 개요

### 한 줄 요약
법을 모르는 일반 시민이 한국 법령 조문을 자연어로 질문하고, RAG 기반으로 정확한 법조문과 함께 답변을 받을 수 있는 AI 법률 챗봇.

### 해결하는 문제
법률 정보를 알고 싶어도 어디서 찾아야 할지 모르고, 법령 검색 사이트는 어렵고, 변호사 상담은 비용이 높다. 일반인이 "내 상황이 법적으로 어떻게 되는지"를 빠르게 확인할 수 없다.

### 핵심 가치
- **조문 직접 인용**: 일반 AI와 달리 실제 법령 조문 번호와 원문을 근거로 답변
- **할루시네이션 방지**: PostgreSQL + Qdrant 이중 검색으로 없는 법을 지어내지 않음
- **한국 법령 특화**: 법령정보원 API 연동으로 최신 법령 자동 업데이트

## 2. 사용자

### 주요 사용자
- **누구**: 법적 판단이 필요한 상황에 처한 일반 시민 (20~50대, 법 비전공자)
- **상황**: 계약서를 검토할 때, 노동 분쟁이 생겼을 때, 교통사고 처리 시, 임대차 문제 발생 시
- **목표**: "내가 맞는 건지, 어떤 법이 적용되는지" 빠르게 확인

### 사용자 시나리오
1. 집주인이 보증금 반환을 거부하는 상황에서 Law11를 열고
2. "전세 보증금 안 돌려주면 어떻게 되나요?"를 입력하면
3. 주택임대차보호법 제3조의2 원문과 함께 절차를 설명하는 답변을 받는다

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
질문 입력 → 라우터 분류 → RAG 검색 (PG → Qdrant → Web) → LLM 생성 → 스트리밍 출력
```

### 상세 흐름
1. **질문 입력**: 사용자가 자연어로 법률 질문 입력
2. **라우터 분류**: `question_router`가 법령/웹/혼합 tool 자동 선택
3. **법령 검색**: PostgreSQL 정확 매칭 → Qdrant 벡터 검색 (threshold 0.45) → 웹 폴백
4. **답변 생성**: GPT-4o가 조문 원문을 근거로 답변 생성
5. **스트리밍 출력**: SSE로 토큰 단위 실시간 렌더링 (status → text → source 순)

### v0.9 이후 흐름 (Multi-turn 추가)
```
세션 확인 → 이전 대화 로드 → 질문 + 컨텍스트 → 라우터 → RAG → LLM → 스트리밍
```

## 5. 성공 기준

### v0.8 (현재)
- [ ] eval 골든 데이터셋 30케이스 정답률 ≥ 70%
- [ ] 할루시네이션 케이스 0건 (eval_hallucination 기준)
- [ ] 라우터 분류 정확도 ≥ 90%

### v0.9+
- [ ] Multi-turn 세션 후속 질문 컨텍스트 유지율 ≥ 90%
- [ ] Citation 표시 후 사용자 신뢰도 지표 상승
- [ ] Reranking 도입 후 top-1 정답률 ≥ 80%

## 6. 안 만드는 것 (Out of Scope)

> Phase별 Out of Scope. AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- **법률 의견 제공** — 이유: 법적 책임 문제. "참고용 정보"만 제공, 면책 문구 필수
- **사용자 회원가입/로그인** — 이유: v0.8 MVP 범위 초과, 세션은 익명으로 관리
- **판례 검색** — 이유: 법령정보원 판례 DB 별도 연동 필요, Phase 3 이후 검토
- **다국어 지원** — 이유: 한국 법령 특화 서비스, 번역 품질 보장 어려움
- **모바일 앱 (iOS/Android)** — 이유: 웹 반응형으로 충분, 앱 배포 오버헤드 큼
